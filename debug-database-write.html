🔍 数据库写入调试工具

此工具帮助诊断为什么支付成功后数据没有写入数据库。

1. 环境变量检查

2. 用户认证状态

3. 数据库连接测试

4. Webhook 配置检查

5. 模拟支付完成处理

6. 问题诊断和解决方案

可能的问题原因：

  1. Webhook 未配置：Stripe webhook 没有正确设置
  2. 环境变量缺失：STRIPE_WEBHOOK_SECRET 未设置
  3. Webhook URL 错误：Stripe 无法访问 webhook 端点
  4. 数据库权限问题：Supabase 权限配置不正确
  5. 事件类型不匹配：Webhook 监听的事件类型不正确

解决方案 1: 检查 Webhook 配置

1. 访问 Stripe Webhooks 设置
2. 检查是否有配置的 webhook
3. 确认 webhook URL: https://yourdomain.com/api/webhook
4. 确认监听的事件: invoice.paid, customer.subscription.updated
5. 检查 webhook 密钥是否正确

解决方案 2: 本地开发 Webhook 测试

1. 安装 Stripe CLI: https://stripe.com/docs/stripe-cli
2. 登录: stripe login
3. 转发 webhook: stripe listen --forward-to localhost:3000/api/webhook
4. 复制 webhook 密钥到 .env.local
5. 测试支付流程

解决方案 3: 手动处理支付完成

在 success 页面添加支付验证逻辑：
1. 验证 Stripe session
2. 获取订阅信息
3. 直接更新数据库
4. 更新用户积分
